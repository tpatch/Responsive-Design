How Responsive Development is Possible

CSS Media Queries

We are able to create styles specifically for different screen sizes.

  • Display the same content differently
  • "Reflow" between different screen resolutions
  • Three columns becomes two, two becomes one

More on Media Queries: W3 Specifications, Mozilla Developer Network

Using Media Queries

  • @media
  • Commonly used for printing
  • Operators: not, and, only
  • only hides styles sheets from older browsers
  • Has several features: color, devices-aspect-ratio, orientation, etc.
  • The device-width feature is used for responsive development

Using Media Queries, cont.

/* Smartphones (portrait and landscape) */

@media only screen and (min-device-width:320px) and (max-device-width:480px)

{

/* Styles */

}


/* iPads (portrait and landscape) */

@media only screen and (min-device-width:768px) and (max-device-width:1024px)

{

/* Styles */

}

Drawbacks and Fallbacks

The good news: Media Queries are widely supported!

The bad news: ...except by IE 6, 7 and 8

You can make your peace with this and build a static size site for old IE, or use a polyfill that adds (limited) Media Query capabilities.


Media Query Polyfills

  • Respond.js
  • Mediatizr

When to Break

Yes, it's likely that the mockups will be designed for specific widths and devices.

But we must be sure that the site doesn't just work for specific devices, but at all sizes.


Two philosophies to setting breakpoints:

  • Set specific breakpoints in advance to target specific devices
  • Set breakpoints in relation to the design

Bottom Up!

The big question: Where to start?

From the bottom (mobile) or from the top (desktop)?

The largest point of contention. It may be easier to start building with the largest layout, but it may be more beneficial to start with the mobile layout.
